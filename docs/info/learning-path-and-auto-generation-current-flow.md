# Learning Path and Auto-Generation: Current Flow

This document describes the **current implemented flow** for:

1. Learning Path task lifecycle
2. Adaptive quiz auto-generation lifecycle
3. Guardrails that prevent same-day quiz chaining after completion

---

## High-Level Architecture

- **DB-driven planning**: `adaptive_study_tasks` is the source for what should be done next (`quiz`, `flashcards`, `review`) and when (`scheduled_date`).
- **Client orchestration**: Learning Path page reads tasks and decides when to trigger quiz generation for tasks with `status = needs_generation`.
- **Quiz completion processing**: quiz attempts update mastery and schedule recalculation, but no direct immediate quiz generation from `useProcessQuizResults`.

## Learning Path Task Flow

## 1) Data loading

On Learning Path load:

- `useAdaptiveStudyTasks()` reads `adaptive_study_tasks` and maps DB statuses:
  - `pending_generation -> needs_generation`
  - `generating -> generating`
  - `ready -> ready`
- `useLearningPathPlan()` combines:
  - adaptive tasks
  - mastery rows
  - documents
  - quizzes

## 2) Task rendering

- `LearningPathCalendar` and `LearningPathContent` render adaptive tasks.
- Quiz task action label in `LearningPathContent`:
  - `ready` -> `Start Quiz`
  - `generating` -> `View Quiz Queue`
  - otherwise -> `Generate Quiz`

## 3) Opening quiz tasks (with baseline reuse fallback)

In `LearningPathContent.handleAdaptiveTaskAction(task)`:

- resolves `effectiveQuizId` as:
  - `task.quizId` first, else
  - fallback to a **ready + unattempted** quiz for the same `documentId`
- if `ready` and `effectiveQuizId` exists -> open `/quizzes/:id`
- if `generating` and `effectiveQuizId` exists -> route to `/quizzes` with highlight
- if `effectiveQuizId` exists but task is not linked yet -> open that quiz directly (prevents duplicate generation)
- if no reusable quiz id:
  - `generating` -> show "still being prepared"
  - `needs_generation` -> call `useGenerateReviewQuiz().mutate(...)`

In `LearningPathCalendar.openTask()`:

- computes `effectiveQuizId` with same fallback rule (`task.quizId` else ready+unattempted quiz by document)
- if `effectiveQuizId` exists, routes into quizzes
- if no `quizId` and `status === generating`, informs user to check quiz queue
- if no `quizId` and not generating, shows "not ready yet" informational toast

## Quiz Completion Flow

When user submits a quiz in `QuizView`:

1. `useSubmitAttempt()` inserts row in `attempts` with `completed_at`
2. `useProcessQuizResults()` runs:
   - writes `question_attempt_log`
   - recomputes concept mastery
   - optionally runs goal-window scheduling
   - invalidates learning/adaptive caches

### Important current rule

`useProcessQuizResults()` **does not directly call** `ensureAdaptiveReviewQuizForDocument(...)` anymore.

This prevents forced immediate generation of a new quiz right after submission.

## Auto-Generation Flow (LearningPathPage)

Current auto-generation trigger lives in `LearningPathPage` `useEffect`.

It looks for `adaptiveTasks` where:

- `task.type === "quiz"`
- `task.status === "needs_generation"`
- no reusable ready+unattempted quiz exists for the same `documentId`
- task not previously generated in this page session (`autoGeneratedTaskIds`)

Then it calls `generateReview.mutate(...)` to create the quiz.

### Same-day guardrail (current behavior)

The effect now computes:

- quiz attempts completed today
- documents that already had a quiz completed today

Then it auto-generates only if:

- `task.scheduledDate > todayLocal`, **or**
- task belongs to a document that has **not** had a quiz completed today

This means:

- New-upload / first-time document tasks can still auto-generate same day.
- After a student completes a quiz for a document today, the system avoids spawning another same-day quiz for that same document.

## Database Sync Flow (adaptive_study_tasks)

`sync_adaptive_study_tasks_for_document(user_id, document_id)` computes adaptive task state using:

- current mastery rows (`user_concept_mastery`)
- existing review quizzes for the document
- quiz attempt history
- flashcards

For quiz tasks, it sets:

- `status` (`pending_generation`, `generating`, `ready`)
- `linked_quiz_id` if a usable quiz already exists
- `scheduled_date` from priority/due logic

Client then maps these fields and decides when to auto-generate if needed.

---

## Review Quiz De-Dup Guard (`useGenerateReviewQuiz`)

`useGenerateReviewQuiz()` now de-dups before invoking the edge function:

1. Reuse existing `Review Quiz:%` with `status = generating` for same document/user.
2. Else reuse latest `Review Quiz:%` with `status = ready` if it has no completed attempt yet.
3. Only then call `generate-quiz`.

This protects against race/overlap between:

- Learning Path auto-effect generation
- user clicking an adaptive quiz task manually

## Expected User-Visible Behavior (Current)

- After upload, baseline quiz can be reused by Learning Path task for the same document (single actionable quiz).
- Learning Path task click should open existing ready+unattempted quiz when available (no forced duplicate review quiz).
- Adaptive auto-generation runs only when no reusable quiz exists for that document task.
- After completing a quiz today for a document:
  - completed quiz appears under completed states
  - same document should not immediately auto-spawn another same-day quiz via page auto-effect
  - subsequent tasks should follow scheduled dates and adaptive plan updates
